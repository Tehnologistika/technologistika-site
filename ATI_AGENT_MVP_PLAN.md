# План реализации MVP ATI-агента ТехноЛогистики

## 1. Введение

Данный документ описывает план создания минимально жизнеспособного продукта (MVP) для AI-агента, автоматизирующего работу с платформой ATI.su. MVP фокусируется на безопасном исследовании возможностей, сборе данных и ручном контроле всех действий, исключая автоматическую публикацию или отправку сообщений без подтверждения.

## 2. Цели MVP

*   Проверить техническую возможность интеграции с API ATI.su и MAX Messenger.
*   Настроить сбор и структурирование данных в Google Sheets.
*   Реализовать базовые сценарии обработки заявок и поиска перевозчиков с ручным подтверждением.
*   Обеспечить полную безопасность и изоляцию от текущих production-систем (включая "боевого Аюба").

## 3. Ограничения MVP

*   **Никаких автоматических действий:** Публикация грузов, отправка сообщений и другие активные действия выполняются **только** после явного подтверждения пользователя.
*   **Изоляция:** Разработка ведется в отдельном репозитории/ветке. Не затрагиваются существующие webhook-и, `.env` файлы, `systemd` сервисы и сайт ТехноЛогистики.
*   **Хранение данных:** В качестве базы данных для MVP используется Google Sheets.

## 4. Этапы реализации

### Этап 1: Подготовка инфраструктуры и доступов

1.  **Создание репозитория:** Создание отдельного приватного репозитория `ati-agent` на GitHub.
2.  **Настройка окружения:** Настройка локального окружения для разработки (Node.js/Python).
3.  **Получение доступов:**
    *   Получение токена для API ATI.su (требуется запрос в поддержку ATI.su для получения `access_token`).
    *   Получение токена для MAX Messenger API (создание бота в MAX).
    *   Настройка сервисного аккаунта Google Cloud для доступа к Google Sheets API.
4.  **Создание Google Sheets:** Создана таблица "ТехноЛогистика: ATI-Agent MVP" (Ссылка: https://docs.google.com/spreadsheets/d/18pMdnkqG7h0B15pR2gWZAH7YeuKKCV-TEYjZsFJvaDw/edit) со следующими вкладками:
    *   Заявки
    *   Перевозчики
    *   Ставки
    *   Переговоры
    *   Рыночная аналитика

### Этап 2: Интеграция с MAX Messenger (Чтение)

1.  **Подключение к API:** Настройка вебхука или long-polling для получения сообщений от бота в MAX Messenger.
2.  **Парсинг заявок:** Разработка логики извлечения данных о заявке (откуда, куда, вес, объем, тип груза) из текстовых сообщений.
3.  **Запись в Google Sheets:** Сохранение распарсенных данных в лист "Заявки" со статусом "Новая".

### Этап 3: Интеграция с ATI.su (Поиск и Аналитика)

1.  **Поиск транспорта (Trucks Search API):**
    *   Разработка функции поиска машин по заданным параметрам (из заявки).
    *   Сохранение найденных перевозчиков в лист "Перевозчики".
2.  **Анализ ставок (Average Rates API / Price Prediction API):**
    *   Разработка функции запроса средних ставок по маршруту.
    *   Сохранение результатов в лист "Рыночная аналитика".

### Этап 4: Реализация ручного подтверждения (Уведомления)

1.  **Интеграция с Gmail:** Настройка отправки email-уведомлений пользователю.
2.  **Сценарий подтверждения:**
    *   При поступлении новой заявки агент формирует черновик для ATI.su и отправляет email с деталями и ссылкой/кнопкой для подтверждения.
    *   При нахождении подходящих перевозчиков агент отправляет email со списком и предложением отправить им запросы.

### Этап 5: Интеграция с ATI.su (Публикация и Сообщения - только после подтверждения)

1.  **Публикация груза (Loads API):**
    *   Разработка функции создания груза на ATI.su.
    *   **Важно:** Функция вызывается *только* после получения подтверждения от пользователя (например, через ответное письмо или клик по специальной ссылке).
    *   Обновление статуса в листе "Заявки" и добавление ATI ID.
2.  **Отправка сообщений (Messenger API):**
    *   Разработка функции отправки сообщений выбранным перевозчикам.
    *   **Важно:** Функция вызывается *только* после подтверждения.
    *   Запись факта отправки в лист "Переговоры".

### Этап 6: Тестирование и отладка

1.  Проведение сквозного тестирования всех сценариев с использованием тестовых данных.
2.  Проверка надежности механизмов ручного подтверждения.
3.  Отладка парсинга сообщений и взаимодействия с API.

## 5. Архитектура MVP

Для MVP предлагается упрощенная архитектура, реализованная в виде единого сервиса (например, на Node.js или Python), который выполняет роль Orchestrator-а и взаимодействует с внешними API.

*   **Язык программирования:** Python (рекомендуется для быстрой разработки скриптов и интеграций) или Node.js.
*   **Хранилище:** Google Sheets API.
*   **Уведомления:** Gmail API.

## 6. План действий на ближайшее время

1.  **Запросить доступы:** Пользователю необходимо запросить `access_token` для API ATI.su и создать бота в MAX Messenger.
2.  **Начать разработку Этапа 1 и 2:** Создать базовый каркас приложения и настроить прием сообщений из MAX.
3.  **Настроить Google Sheets API:** Интегрировать скрипт с созданной таблицей.

## 7. Критерии успеха MVP

*   Агент успешно получает заявки из MAX Messenger и записывает их в Google Sheets.
*   Агент способен находить перевозчиков на ATI.su по заданным параметрам и сохранять их в таблицу.
*   Агент запрашивает рыночные ставки и сохраняет аналитику.
*   **Ни одно действие на ATI.su (публикация, сообщение) не выполняется без явного ручного подтверждения.**
*   Существующие системы ТехноЛогистики не затронуты.
